# 結局清單篩選與實裝規劃

依據 2026-07 提供的結局擴充清單（A～I 共 70+ 個）進行篩選。

## 篩選標準

1. **現在就能做**：條件可以用現有系統判定（六個數值＋卡片使用次數），且素材齊全（Demo 的 5 張卡、1 位女主、3 個場景）
2. **等內容到位再做**：需要尚未實裝的卡片（布偶裝、領域展開等）、女主角、章節或場景
3. **建議合併**：概念重疊的結局合併成一個，避免圖鑑灌水
4. **建議降級為「事件」**：點子很好，但更適合當劇情事件而非結局
5. **建議做成「隱藏彩蛋」**：第四面牆類，當結局會傷害代入感，當彩蛋是驚喜
6. **建議捨棄／緩議**：依賴不存在的系統或卡片

## 第二批：強烈推薦，等對應內容實裝（優先做）

### 全破級大結局（I 類）
- **普通人的戀愛** ⭐ 最推薦的「真結局」——不用任何卡片、用自己的話告白。與 GDD 第 13.5 節「真・Gumayuwei 結局」精神一致，建議兩者合併：外殼用「真・Gumayuwei」名字，內容用「普通人的戀愛」演出（需要「不出牌」這個特殊選項的系統支援）
- **梗圖之神**：全收集的第二真結局，與「卡片圖鑑完成」合併（同為全收集獎勵，留一個）

### 社死成功系（B 類，本遊戲的招牌特色，優先度高）
- **全公司都在看**：誤開麥克風告白（告白章實裝時做）。「午休廣播告白」與它概念重疊 → 合併
- **主管也嗑到了**：主管已在 Demo 登場，第一章擴充時就能做，成本低
- **電梯門開了**：電梯場景已存在，且能回收「我還沒上車啊」哏，成本低
- **茶水間傳說**：需要「領域展開」卡，第三章後做
- **公司群組爆紅**：與「社死失敗」結局重疊度高 → 建議合併進化：社死失敗保留為壞結局，群組下注哏移進它的結局文字

## 第三批：做成「隱藏彩蛋／成就」而非結局（第四面牆類）

這些很好笑，但當正式結局會打斷戀愛主線的代入感，建議做成**隱藏彩蛋**（特殊條件觸發的一次性演出＋成就登錄）：

- 戀愛 Debug（女主要求看好感度公式）
- 主角本人拒絕營業（Gumayuwei 看鏡頭嗆玩家）
- 下週三維護補發（告白失敗的系統公告）
- AI 生成失控（立繪隨機亂變——技術上超好做，笑果好）
- 卡片反噬（所有卡片對他說「不急」——適合當全收集彩蛋）
- Claude Code 結局／Codex 本地端結局（工程師內梗，受眾較窄，放成就名／彩蛋文字最安全）

## 第五批：緩議／捨棄

- **薪水被偷**：依賴不存在的「偷薪水小偷」卡 → 除非要新增該卡，否則捨棄
- **台北車站迷路**：需要「走錯出口」的玩法系統（迷宮小遊戲），成本高 → 緩議，或降級為一個約會事件
- **夜市制霸／不能吃太胖／加餐原始人**：都需要食慾值系統與對應卡片 → 隨第二章夜市場景一起評估，先留「夜市制霸」一個代表
- **摩艾夜跑／紅色植物大量繁殖／自由女神搬到台北／鼻子壞掉連鎖／副本變婚禮**：超展開系保留 2 個就夠（推薦：**貓咪收編**、**不急時間停止**，上面已列），其餘降級為事件演出素材，避免結局圖鑑太稀釋
