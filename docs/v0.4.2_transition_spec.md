# Guma_Game v0.4.2 轉場升級規格

## 版本目標

v0.4.2 的目標是讓目前的 HTML / CSS / JS 原型具備更像正式 AVG / 戀愛模擬遊戲的演出質感。

本版採用「外掛式升級」策略：

- 不重寫 `main.js` 主流程
- 不修改 JSON 劇情資料
- 透過 `v042-transitions.css` 與 `v042-transitions.js` 包裝既有流程
- 降低再次出現流程卡住的風險

---

## 新增檔案

```text
prototype/v042-transitions.css
prototype/v042-transitions.js
```

`index.html` 已加入：

```html
<link rel="stylesheet" href="v042-transitions.css" />

```

## 轉場設計原則

### 1. 動畫節奏

```text
快速 UI 回饋：180ms ~ 300ms
一般轉場：300ms ~ 420ms
場景切換：620ms ~ 780ms
重要演出：700ms ~ 920ms
```

### 2. easing

本版新增兩組高級感曲線：

```css
--ease-premium: cubic-bezier(0.22, 1, 0.36, 1);
--ease-snap: cubic-bezier(0.16, 1, 0.3, 1);
```

用途：

- `--ease-premium`：場景、背景、對話框、結局標題
- `--ease-snap`：卡片、名牌、抽卡翻牌

## 已實作動畫

### 1. 場景轉場 Scene Wipe

觸發：`loadScene(sceneId)`

效果：

1. 全螢幕暗色轉場層淡入
2. 顯示 `LOADING NEXT SCENE`
3. 背景切換
4. 轉場層淡出

檔案：

```text
#transition-wipe
@keyframes sceneWipe
```

---

### 2. 背景進場

觸發：新場景載入後

效果：

- 背景先微放大
- 模糊
- 暗化
- 再清晰回到正常比例

檔案：

```text
.bg.scene-entering
@keyframes bgPremiumIn
```

---

### 3. 角色進場

觸發：新場景載入後

效果：

- 男主從左側滑入
- 女主從右側滑入
- 同時淡入與去模糊

檔案：

```text
.sprite.scene-enter-left
.sprite.scene-enter-right
```

---

### 4. 對話框進場

觸發：`renderLine(line)`

效果：

- 對話框往上浮入
- 由模糊變清晰
- 名牌小幅彈出

檔案：

```text
.dialog.dialog-enter
.nameplate.nameplate-pop
```

---

### 5. 選卡區進場

觸發：`openHand()`

效果：

- 選卡區淡入
- 背景略微 blur
- 上方對話參考框浮入
- 卡片由下方依序展開

檔案：

```text
.hand-overlay.hand-enter
.choice-context
.hand .card
@keyframes cardFanIn
```

---

### 6. 選卡確認動畫

觸發：`playCard(cardId)` 前

效果：

- 選中的卡片向上浮起、放大、發光
- 其他卡片變暗、灰階、模糊
- 延遲約 260ms 後才進入原本出牌流程

目的：

讓玩家感覺「這張卡被正式選中了」。

---

### 7. Cut-in 強化

觸發：`showCutin(card, effects, bonus, onDismiss)`

效果：

- 保留原本速度線
- 卡片出現時增加金色 pulse
- 強化出牌爽感

檔案：

```text
.cutin.v042-cutin-pulse
@keyframes cutinPulse
```

---

### 8. 抽卡翻牌感

觸發：抽卡結果卡片渲染時

效果：

- 卡片從 rotateY 角度翻入
- 帶一點上浮與回彈

檔案：

```text
.gacha-card
@keyframes drawReveal
```

---

### 9. 結局標題演出

觸發：`showEnding()`

效果：

- 結局標題模糊淡入
- 字距收斂
- 下滑進場

檔案：

```text
.ending-title
@keyframes endingTitleIn
```

## 實作方式

`v042-transitions.js` 使用 patch 包裝方式：

```js
patchFunction("loadScene", ...)
patchFunction("renderLine", ...)
patchFunction("openHand", ...)
patchFunction("playCard", ...)
patchFunction("showCutin", ...)
patchFunction("openRunDraw", ...)
patchFunction("showEnding", ...)
```

優點：

1. 不直接重寫主流程
2. 出問題時可以快速移除 v0.4.2 檔案
3. 主邏輯仍由 `main.js` 控制
4. 未來轉場可獨立調整

## 卡片素材建議規格

### 原始卡片模板

```text
1536 × 2048 px
比例 3:4
格式：PSD / Figma / PNG master
```

### 遊戲輸出尺寸

```text
768 × 1024 px
格式：webp
品質：80 ~ 90
```

### 卡片結構

```text
上方：稀有度角標
中間：主圖區
下方：卡名 / 台詞 / 簡短描述
邊框：依稀有度變化
```

### 建議流程

```text
原始卡圖
→ 放入 Figma 3:4 卡片模板
→ 套固定遮罩
→ 套稀有度邊框
→ 輸出 webp
→ 放入 assets/cards
```
